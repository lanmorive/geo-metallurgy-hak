# ТЗ: «Научный клубок» (Норникель AI Science Hack, трек 2)

Создание поисково-аналитической системы для выявления связанных сущностей:
статей, экспериментов, материалов, свойств, режимов, установок,
исследовательских команд — в специализированной литературе и научных данных
горно-металлургической отрасли.

## Проблема заказчика
- Знания R&D разбросаны по тысячам отчётов, статей, патентов (RU + EN)
- Поиск по ключевым словам не находит связи «материал → процесс → режим →
  результат → команда»
- Литобзоры делаются вручную неделями, дублируются между командами
- Институциональная память теряется с уходом экспертов

## Функциональные требования
1. **Импорт документов**: PDF, DOCX; русский и английский языки
2. **Извлечение сущностей** по онтологии (см. `docs/ONTOLOGY.md`):
   материалы, процессы, оборудование, свойства, эксперименты, публикации,
   эксперты, установки/площадки
3. **Извлечение связей** между сущностями, включая противоречия
   между источниками
4. **Извлечение числовых ограничений**: концентрации, температуры,
   давления, скорости потоков — с единицами измерения.
   Пример из ТЗ: «сульфаты ≤ 300 мг/л». Ошибки в числах недопустимы.
5. **Семантический поиск на естественном языке** (RU/EN) с учётом
   синонимии терминов между языками
6. **Фильтры**: география (Россия/мир), год, уровень достоверности,
   числовые диапазоны параметров
7. **Синтез структурированного ответа** типа «литобзор»:
   - группировка по методам / годам / географии
   - ссылки на источники у каждого утверждения
   - консенсус и противоречия между источниками
   - уровень уверенности
8. **Выявление пробелов знаний** — комбинации условий, по которым нет
   экспериментов/публикаций
9. **Визуализация графа** связей с возможностью навигации
10. **Верифицируемость**: каждый факт → источник, confidence, дата, география

## Нефункциональные требования
- Масштабируемость до ~1 млн сущностей (на хакатоне — архитектурно,
  словами в презентации)
- Расширяемость на новые домены (новая онтология без переписывания кода)
- Ролевая модель доступа (на хакатоне — заглушка)

## Эталонные запросы жюри (демо строим вокруг них!)
Точные формулировки — в `docs/DEMO.md`. Тематики:
1. Обессоливание воды при заданных диапазонах сульфатов и сухого остатка
2. Поиск экспериментов по процессу с фильтрами по условиям
3. Связи материал → свойства → режимы обработки
4. Поиск экспертов/команд по теме

## Данные
Корпус с Яндекс.Диска организаторов → положить в `data/raw/` (в .gitignore):
отчёты и статьи RU/EN, каталог экспериментов, справочники материалов
и оборудования, список экспертов, таксономия тегов.
